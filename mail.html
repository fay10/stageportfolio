E-mail signature

Betrokken bij dit project zijn: Cyd, Merijn,

Dit was mijn eerste officiële opdracht. Merijn heeft een design voor me gemaakt die ik kon coderen. De bedoeling was dat werknemers van Matise hun e-mail signature gemakkelijk konden aanmaken, met behulp van een e-mail signature generator. Tijdens deze opdracht heb ik voor het eerst kunnen ervaren hoe de developers samenwerken binnen Matise.

Ik heb namelijk voor het eerst branches gemaakt en ik kom er nu achter dat GitHub erg handig is, vooral voor bedrijven met developers. Ik heb ook voor het eerst gewerkt met Jira.

Cyd heeft de website opgedeeld in verschillende userinterface blokken. Van die userinterface blokken heeft ze weer aparte tickets gemaakt. En zoals ik al verteld heb werken Jira en Github met elkaar.

Na elke ticket die ik af had moest ik mergen naar develop en daarna weer een nieuwe branch aanmaken. Na elke nieuwe branch die ik aangemaakt had ging Cyd op Github mijn code checken.

Werken met Github begreep ik nog niet zo goed, omdat ik niet goed wist wat hiervan de bedoeling was. Wat ik wel begreep is dat developers elke keer hun stukje code (branch) naar de main branch sturen. Op deze manier kunnen ze elkaars codes verbeteren en naar elkaar pushen.







Ik begon met het de header, De window icons heb ik met een zelfgemaakte array gemaakt. Het gedeelte links van de preview dat lijkt op een echte inbox van je e-mail heb ik ook met behulp van een array gemaakt. Array’s maken vind ik niet meer zo moeilijk nu en ik denk dat ik het nu ook goed begrijp.

De e-mail signature preview en het formulier was wel te coderen omdat ik dezelfde opzet van de opdracht Form kon gebruiken. Het formulier zelf en de preview hadden wel andere codes. Ik dacht dat de rest super moeilijk zou zijn maar gelukkig was een groot deel van de functies in de websites getackeld door de plugins. Het punt is alleen dat je deze plugins goed moet begrijpen voordat je deze wilt gebruiken.

Tijdens deze opdracht ben ik namelijk bezig geweest met de plugins vee-validate, v-clipboard, v-mask en v-prism. Passend bij deze plugins heb ik veel gebruik gemaakt van data en methods.

Vee-validate is een validator plugin voor je formulier,

V-clipboard pluginzorgt ervoor dat je bepaalde teksten en plaatjes kan kopiëren

V-mask zorgt ervoor dat je alleen bepaalde informatie in een input kan typen

V-prism verbergt elementen en zorgt ervoor dat bepaalde delen van je website alleen zichtbaar worden wanneer je op een bepaalde knop klikt bijvoorbeeld.

Voor het fomulier heb ik opnieuw de vee-validate plugin gebruikt. Dit verliep vrijwel soepel. Ook omdat ik al eerder ervaring heb met deze plugin en ik het de eerste keer niet goed begreep maar Cyd heeft deze plugin samen met de andere nog een keer uitgelegd.

Daarnaast wil je dit visuele gedeelte of de code ook kopiëren om in je e-mail te stoppen. Dit heb ik gemaakt met de plugin v-clipboard en een v-if statement en met methods. Ik heb erg veel geleerd vooral op gebied van plugins, en ik ben echt verbaasd hoeveel je ook kan met Vue en Nuxt. Een deel van de plugins begrijp ik, dus dat houdt ook in dat ik de data en functions ervan begreep. Van de plugin v-mask en en v-prism begrijp ik alleen niet zoveel.

Nadat ik mijn de website af had heeft Cyd laten zien hoe Netlify werkt. Netlify is een erg handige tool waarmee je websites van Github makkelijk online kan zetten. Eerlijk gezegd dacht ik dat de website klaar was.

Toen de website eenmaal online stond hebben er nog andere designers en developers naar gekeken en de website getest. Zij hebben via Jira dan weer bugs aangegeven waar Cyd tickets van heeft gemaakt.

Er zat bijvoorbeeld een bug in die alleen te zien was op een Windows laptop. Om te kunnen zien wat de bug van de website op een Windows laptop was had Cyd mij een tool gegeven genaamd Browerstack.

Het gedeelte van bugs oplossen hoort natuurlijk ook bij front-end development. Het testen van je website op elke versie en apparaat mogelijk is dus ook erg van belang heb ik geleerd. Een website is dus pasklaar wanneer het op alle apparaten werkt.

Visit Aalsmeer

Betrokken bij dit project zijn: Cyd, Seb, Jantine

Dit is mijn eerste klantopdracht. Dit houdt in dat ik mijn tijd moest tracken via Harvest. Werken met Harvest is vrij simpel. Je kiest het project waar je mee bezig bent en geeft aan of je front-end developer, back-end of designer bent. De website Visit Aalsmeer is een toeristen website. De eerste wens van de klant was om de website aantrekkelijk te maken voor buitenlandse toeristen door de website ook in het Engels mogelijk te maken.

De start

Als je aan een nieuwe website begint moet je die eerst binnenhalen van Github. In de readme.md staat dan op welke manier. Voor deze website moest je vagrant, composer, grunt en valet gebruiken. Cyd heeft me geholpen om al deze tools te instaleren op mijn computer.

Vagrant: dit is een tool/framework die je kan gebruiken als je gebruik maakt van virtual box. Virtual box is een soort vervangende versie van Xampp of Mamp.

Composer: composer is een dependency manager voor PHP. Dit is een soort van package.json maar dan compose.json.

Grunt: Grunt is een task runner en zorgt voor de SCSS en javascript compiling en minifying.

Valet: Valet is een lokale database is en dat het vooral bedoel is voor ontwikkelen en testen.

Ik heb van Seb een database gekregen of te wel een MySQL bestand. Cyd heeft voorgedaan hoe ze een database in het project integreren. Dit heeft ze gedaan met behulp van Valet. Dit gedeelte vond ik erg moeilijk aangezien ik nog niet precies wist wat valet en MySQL inhield en ook omdat dit compleet anders is dan PHP, javascript, vue of nuxt.





Daarna heb ik met behulp van git clone de website op mijn computer gezet. In het begin was het wel een uitdaging om aan iemand anders code te werken. Omdat je zelf niet alle content in de website hebt gezet ook was de opbouw van de code anders.

Ik heb de opdracht gekregen om een language switcher te maken voor hun website. Ik heb eerst de switcher gestijld volgens het gewenste design en daarna heb ik de functie van de switcher gecodeerd met behulp van een PHP plugin genaamd polylang.

Deze plugin moest in de composer.json staan. Net zoals je javascript of vue plugins in je package.json bestand zet. Nu begreep ik ook het verschil en de overeenkomst van composer en packages.

Cyd heeft me een website gegeven waar de documentatie van de PHP plugin opstond. Ik begreep nog vrij weinig van deze PHP codes. Ook was dit de eerste keer dat ik gewerkt heb met PHP. Met PHP werken vind ik erg moeilijk, vooral omdat je hiervoor echt de code moet begrijpen wil je weten wat je aan het doen bent. Gelukkig heeft Cyd me geholpen en uitgelegd hoe alles werkt.

Na de language switcher ben ik aan de slag gegaan met de tweede wens van de opdrachtgever. Dde klant wilde de foto’s hoger op hun website, en gecentreerde video’s. de video's heb ik gemaakt met behulp van een WordPress player. De foto’s hoger en te centreren lukte wel in mijn eentje alleen duurde me dit even. Ik moest namelijk de foto’s op elke pagina wijzigen en controleren. Omdat de code van de website dus geschreven was door iemand anders duurde het even om elke classnaam van elke foto te vinden. Ook deed ik er super lang over om de foto’s op mobiel formaat en op elke browser ondersteuning goed te krijgen.

Jantine had deze week een stand-up ingepland. Dit vond ik erg fijn want dan kon ik samen met Cyd vertellen wat we hadden gemaakt, wat er fout ging en wat er nog gemaakt moet worden.

Wat wel even een tegenslag was voor mij was toen de foto’s die ik met SCSS gemaakt had niet goed genoeg waren. Ik deed er juist zo lang over om ze te centreren en ze op elke pagina juist te krijgen. Gelukkig is Cyd doorgegaan met de foto’s. Ik ben hierna verder gegaan met de video’s.

De video in de website zetten viel op zich wel mee. Ik heb voor dit gedeeltde custom fields gebruikt. Voordat ik begon met het maken van het tweede gedeelte van Visit Aalsmeer heb ik gewerkt aan kindergarden. Met kindergarden heb ik ook gewerkt met custom fields dus gelukkig kwam dit gedeelte me al bekend voor.

Ik heb twee slides gemaakt. De eerst slider is geschikt voor filmpjes en foto’s die de klant dynamisch met behulp van custom fields in de website kan zetten.

De tweede slider is alleen geschikt voor foto’s en is zichtbaar wanneer je de website op mobiel scherm bekijkt. Deze slides heb ik toegevoegd met PHP. Ik begreep deze PHP code goed ook omdat dit gewoon gemaakt werd met een if en een else en ik daar al wat ervaring mee heb.
