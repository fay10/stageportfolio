De mensen van Matise



  

Bereid voor inovatie

,

Over Matise: Matise is een digital agency en is van een klein bedrijf naar een bedrijf gegroeid met dertien werknemers. Matise ontwikkelt websites, apps, en webshops. Matise is een bedrijf dat zich inzet voor kwaliteit en onderneming. Matise zit aan het Entrepotdok tegenover Artis. Vanuit het kantoor kan je zo het olifantenverblijf zien, en met een beetje geluk zie je soms ook de giraffen.

Matise is ook erg bezig met constante ontwikkeling. Thomas en Seb gaan bijvoorbeeld naar een maandelijkse cursus om hun bedrijf te verbeteren. Er wordt soms ook een let’s talk gegeven. Hierin vertellen werknemers wat zij geleerd hebben en delen dit dan met de rest van de groep. Ook gaan werknemers soms met elkaar naar beurzen of congressen. Ik vind dit een erg goed initiatief, dit laat al zien dat Matise ook buiten hun bedrijf continu opzoek is naar nieuwe informatie.

Werkzaamheden binnen Matise

Matise begint de ochtend meestal met tien minuten stand-up. Hierin legt iedereen in het kort uit waaraan hij/zij de komende dag aan gaat werken. Na twee weken stage lopen heeft Roosje de stand-up wat aangepast. Seb en Roosje wilde de stand-up namelijk wat globaler maken. Vanaf nu is de stand-up alleen op maandag gepland. De stand-up duurt meestal een kwartier.

Tijdens de stand-up bespreken we dan algemene updates van Matise zelf. Ook wordt er gekeken naar de successen van vorige week en wordt de agenda van aankomende week besproken. De klant gesprekken en klant bezoeken worden ook besproken.

Matise houdt voor elk project ook vaak een stand-up, tijdens deze stand-up bespreek je dan kort met elkaar waar iedereen staat in het project en wat er gedaan moet worden. Hoe vaak de stand-up is hangt af van het project zelf.

Nieuwe klant Matise


Een klant komt voor het eerst bij Matise binnen via Tim, Maurits of Seb. Wanneer er een klant wordt binnengehaald wordt Thomas er al snel bijgetrokken. Hij gaat dan overleggen met de klant wat de klant precies wil en wat de doelgroep van de klant is. Thomas probeert dan niet de oplossing van het idee van een website te achterhalen maar het probleem.

Gebaseerd hierop gaat Seb, Thomas of het projectmanagement team dan aan de slag met een offerte. Meestal is er na een gesprek met Thomas al snel duidelijk wat de klant wil en dan is het vanzelfsprekend wat er ook gemaakt moet worden.

Passend bij een gemaakte offerte maakt Seb, Thomas of Roosje meestal een projectplan. Hierin wordt uitgelegd waar de gewerkte uren aan besteed gaan worden. De planning en doelen worden in het projectplan aangegeven. Ook wordt er een globale tijdlijn aangegeven. Zo weet de klant en Matise zelf waar iedereen aan toe is.



Nadat het projectplan en de offerte gecontroleerd is door beide partijen kan Roosje of Jantine beginnen met een kick-off. De designers en/of developers gaan dan samen bespreken wie wat aan de website gaat doen en of dat ook haalbaar. Wanneer Matise een nieuwe klant krijgt organiseert de projectmanager een kick-off. Tijdens een kick-off bespreekt Jantine dan bijvoorbeeld alle onderwerpen die aan bod komen bij het maken van een product voor de klant.

Daarna wijst de projectmanager de juiste persoon aan voor de juiste taak. Een kick-off verschilt erg per klant. Tijdens het begin van een nieuw project wordt er eerst een algemene kick-off gehouden. Daarna maakt de projectmanager kleine vergaderingen aan die van belang zijn voor het designteam. Wanneer de projectmanager de taken onderling de designers heeft verdeeld gaan de designers aan de slag.

Als de designers klaar zijn en hun ontwerp is gecontroleerd door de klant, begint de projectmanager met een development kick-off. Hierin bespreekt de project manager dan opnieuw alle taken maar dan alleen op development gericht. Wanneer de opdracht af is, is er nog een laatste evaluatiegesprek waarin Matise en de klant hun project afronden.



Werkwijze binnen Matise


Sketch is een design programma waar de ontwerpers van Matise mee werken. Via Sketch worden de designs gemaakt. Sketch wordt alleen gebruikt door de designers.

Zeplin is bedoeld voor zowel designers als developers en klanten. Als designers hun design gemaakt hebben uploaden ze dit design naar Zeplin. Developers en soms klanten kunnen dan meekijken en feedback geven. Developers kunnen dan ook fonts, pixels, en kleuren van de website overnemen om ze vervolgens in hun code te zetten.

Slack wordt gebruikt als werk gerelateerd communicatiemiddel binnen Matise. Er worden dan verschillende groepen aangemaakt, aansluitend bij het project waarin werknemers mee werken. Er worden via Slack bijvoorbeeld bestanden gedeeld of andere belangrijke informatie.

Jira wordt gebruikt voor tickets en is een projectmanagement tool die Matise helpt om alle functionaliteiten en bugs bij te houden. Als developers gaan coderen, coderen ze ook niet direct de hele website. Dit doen ze in delen, oftewel userinterface blokken. Zo kunnen ze gestructureerd werken. Ook wordt Jira volop gebruikt bij bugs en de testing van een website.

Abstract is een beetje hetzelfde als GitHub maar dan voor de designers. In dit programma kunnen designers hun design ook mergen en pushen.

GitHub wordt gebruikt voor developers binnen Matise. Github is een versie beheer Tool voor Matise. Via GitHub kunnen developers elkaars code inzien en onduidelijkheden in de code vermelden. Wanneer de officiële gecodeerde website af is, wordt deze via GitHub eerst nog getest en daarna online gezet.

Harvest wordt gebruikt om bij te houden hoeveel uur iemand aan een bepaald project werkt. Tijd bijhouden is erg belangrijk voor projectmanagement maar ook voor jezelf kan dit erg van toepassing zijn. Wanneer je je tijd hebt bijgehouden en ingevoerd hebt maakt Roosje of Jantine daar een passende factuur bij voor de klant.

Codeship deployed je eigen code. Codeship werkt samen met GitHub. Codeship zorgt voor de integratie van je code en de delivery van je code. Codeship laat zien wat er in de code is toegevoegd aan je project en welke terminal commands er werden uitgevoerd.

Werkwijze developers





Developers binnen Matise krijgen een design en delen die eerst op in userinterface blokken. Van die blokken maken ze dan tickets met behulp van Jira.

Deze tickets krijgen dan allemaal hetzelfde nummer (code) met daaropvolgend een verschillende naam. Als je de tickets hebt aangemaakt met je UI-blok en maak je daar een passende branch op github (branch is nieuwe versie code van de website) bij. Die branch naam moet wel dezelfde code hebben als je ticket.

Matise GitHub is namelijk gekoppeld met Jira dus als je een nieuwe branch aanmaakt push je deze direct naar GitHub zodat het team in Jira kan zien dat je met dit ticket bezig bent. Die branches die maak je aan ook omdat andere collega's dan naar je werk kunnen kijken en fouten of bugs kunnen vinden in je code. Wanneer alles gecontroleerd is kan je je branch mergen met je main branch genaamd develop. Zo werkt iedereen aan dezelfde website maar wel aan een ander stukje UI dus.



Matise werkt met drie main branches

- Dev: De dev branch omgeving is bedoeld voor de developers om op te werken en telkens nieuwe features (branches) weer toe te voegen.

- Staging: De staging branch is gekoppeld aan een staging website waar klanten naar kunnen kijken. Op dit moment staat de branch nog NIET online, wel staat de website online voor de klant.

- Master : De master branch is de laatste branch. Als je website helemaal af is en iedereen alles gemerged heeft kan je je develop branche mergen met de master branch. Die master branch wordt vervolgens met Netlify gedeployed of je kan je website eventueel online zetten via rsync.

Wanneer iedereen klaar is met een onderdeel van de website (ui blok) gaan uiteindelijk de designers ook nog eens goed of alle pixels en font op maat zijn en kloppen. Wanneer dit het geval is kan de website online en naar master gepusht worden

Slot: Ik vind de manier van werken bij Matise erg gestructureerd en handig. Ik vind dat de manier van klanten opdrachten goed aanpakken. Ook is Matise continue bezig met elkaar te verbeteren.

Er worden ook vaak gezamelijke brainstorms gegeven en iedereen probeert elkaar te helpen. Ook hebben we een maandelijkse borrel.
