Inleiding

De eerste opdracht was kennismaken met de manier waarop de front-end developers coderen. Ik ga eerst vertellen hoe front-end developers werken en daarna vertel ik over de syntax die de developers gebruiken.

De eerste week heb ik mijn intake opdracht opnieuw gemaakt. De opdracht was om een website te coderen volgens het gemaakte design. Ik heb deze opdracht opnieuw gemaakt, maar dan met de tools die Matise gebruikt om een website te maken.

Ik heb daarnaast kennis gemaakt met een framework genaamd Vue. Daarnaast werkt Matise met Nuxt, deze plugin zorgt voor de server rendering voor hun website en helpt bij de routing. Werken met Vue had ik wel overschat. Ik merk dat Vue eigenlijk best makkelijk is om te begrijpen. Dit kwam waarschijnlijk ook, omdat ik voor mijn stage al wat onderzoek had gedaan naar Vue.

Ik heb tijdens deze opdracht ook voor het eerst gewerkt met Henris. Henris bestaat uit een grid van vierentwintig blokken. Eén grid is dan tachtig pixels op een scherm van 1920px.




Ik heb ook geleerd hoe Henris op zijn eigen manier fonts en kleuren importeert. Werken met dit framework was wel te doen het is meer dat je er gewoon vaker mee moet werken. Omdat je bijvoorbeeld niet zomaar de pixels van bepaalde elementen kan typen, je moet die pixels dan weer omzetten naar grid maten, dit ging eerst nog niet vanzelf bij mij. Ook had ik nog de neiging om op de manier die ik geleerd had op school fonts en kleuren binnen te halen.

Werken met SCSS en BEM was een uitdaging omdat je je code precies moet schrijven volgens deze regels.

Dit houdt in dat je HMTL code in het begin al een goed skelet moet hebben volgens de BEM regels.

Dit betekent dat je goed moest begrijpen hoe je eigen code in elkaar zat voordat je ze kan definiëren in je SCSS. Hier had ik in het begin wel echt moeite mee. Ook omdat ik nooit echt codes begreep. Ik heb hier ook veel van geleerd.



Cyd heeft laten tijdens deze opdracht laten zien hoe components en sections werken: In het elements mapje staan vaak elementen die je vaker dan één keer op je website wilt gebruiken. Meestal zijn dit buttons.

In het mapje layout staat de header en de footer. Soms ook de sidebar in het mapje sections staan grotere elements die je vaker wil gebruiken in een pagina.

Ook heb ik geleerd hoe je specifieke data van een api in je website kan integreren. Dit onderdeel vond ik wel erg pittig. Tijdens de uitleg van dit onderdeel begreep ik er relatief weinig van omdat ik maar één keer gewerkt hebt met Api’s en nog nooit met data.

Wat ook fijn was is dat Cyd wel gewoon 2 dagen intensief bezig was met het helpen bij deze opdracht. Ze heeft ook veel uitgelegd waar ik later gewoon veel aan had. Oefenen met Henris en flexbox




Tijdens deze opdracht heb ik wat meer gewerkt met flexbox. Flexbox was nog vrij nieuw voor mij en het was wel van belang om flexbox te beheersen Cyd zei namelijk ook dat matise veel gebruik maakt van flexbox en containers.

Ook brengt flexbox veel andere voordelen met zich mee. Flexbox zorgt er bijvoorbeeld voor dat je websites makkelijker responsive kan maken. Met flexbox kan je ook eenvoudiger je elementen plaatsen in je website.

Ik heb een design gemaakt met blokken en dit gecodeerd. Ik heb tijdens deze opdracht ook weer geoefend met SASS dat vond ik ook erg fijn want SASS en BEM moet ik uiteindelijk zonder moeite toepassen.

Cyd had zelf niet veel tijd om me te helpen. Daar had ik zelf wel moeite mee omdat dit nog steeds wel allemaal nieuw was voor mij. Toen Cyd merkte dat ik met flexbox nog wat moeite had heeft ze me een link gestuurd waarbij je een soort interactief spel speelt en je alleen naar een volgend level kan als je de juiste code intypt. Met behulp van deze link heb ik vooral veel geleerd.

Ook begrijp ik meer van containers omdat je deze nodig hebt wil je gebruik maken van flexbox. Ik ben ook blij dat ik nu weet hoe je op de beste manier je website responsive kan maken. Ik heb geleerd dat flexbox helemaal niet zo moeilijk is, als je het maar goed begrijpt.

Service Level Agreement (SLA)




Matise biedt hun klanten wekelijks of maandelijks een service level agreement. Dit houdt in dat ze de plugins van hun WordPress website updaten. Ook krijgen de websites een malware scan, wordt er spam berichten verwijderd en worden er back-ups gemaakt. Daarnaast controleert Matise of de website nog helemaal werkt en of er geen bugs zijn. Sinds ik bij Matise werk ga ik elke dinsdag aan de slag met de SLA.

Sommige plugins werken niet goed samen met bepaalde codes in de website. Hierdoor gaat de website soms kapot. Dit kan je dan controleren door te kijken of de website nog overeenkomt met het design en of de javascript van de website het nog doet.

Als er een website vastloopt na het updaten dan moet je het project binnenhalen en de bugs oplossen. Ik had weleens een bug gevonden in de website aangenaam in business. De bug was dat de pagination het niet deed. Er zat blijkbaar een fout in de PHP code. Dit was niet een hele grote fout, de php code mistte namelijk gewoon een groot gedeelte. Cyd heeft me hiermee geholpen want het had te maken met de pagination en de aantal post_per_page function.

Tijdens deze opdracht heeft Cyd een design voor mij gemaakt waar ik later aan kon werken. Voor deze slider heb ik een plugin gebruikt die ze ook bij Matise zelf ook toepassen op hun website.

De plugin heet io dangerous swiper. Plugins download je eerst met npm, daarna zet je ze in de package.json, ook maak je een apart javascript bestand waar je deze plugin import. Na elke plugin die je in je website zet moet je npm run dev opnieuw draaien om je website up te daten. Dit gedeelte vond ik niet erg moeilijk.

Ik ben met deze opdracht ook aan de slag gegaan met data en functies dit vond ik wel erg moeilijk maar Cyd heeft wel groot een deels uitgelegd hoe data en functies werken.

Functies gebruik je als javascript classes en de data zet je in onder elkaar met een true of false functie van de plugin zelf. Ik begrijp hier nog niet veel maar ik begrijp al wat meer van functions en data dan voorheen.
