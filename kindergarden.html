Kindergarden

Betrokken bij dit project zijn: Cyd, Seb, Merijn, Jantine en Roosje

Dit was de eerste opdracht waar ik aan meegewerkt heb van begin tot einde. De klant wilde een update voor haar website. Matise begint een opdracht met een kick-off. Deze kick-off werd gegeven door Jantine. Jantine had samen met Stephan een lijst gemaakt waar alle taken in stonden met daarin de geschatte uren. Daarna heeft Jantine deze taken verdeeld tijdens de allereerste kick-off.

De website bestaat uit blokken met tekst en foto’s, met soms een extra passend blok waarin een quote staat. Mijn eerste taak was om de nieuwe blokken in de website te zetten, want de oude blokken waren niet geschikt voor de website.

Het binnenhalen van dit project verliep tot mijn verbazing gewoon soepel. Ik heb git clone gebruikt. Ik had wat problemen met de versie van node maar Cyd heeft uitgelegd dat ik gewoon nvm use 8 kan gebruiken om terug te gaan naar de versie die kindergarden gebruikt. De website van Kindergarden is gemaakt in WordPress. Om een groot deel van de website te maken had Sil een plugin gedownload genaamd custom fields. Dit is een plugin waarin je velden kan aanmaken en zo een pagina dynamisch kan inrichten.

Custom fields biedt ook de mogelijkheid om blokken aan te maken. Sill had de custom fields blokken aangemaakt maar het was handiger als die blokken anders werden ingedeeld. Daarom heb ik nieuwe blokken gemaakt die voor de klant dynamischer waren maar ook kon Seb nu makkelijker deze blokken in de back-end zetten. Ik heb een pdf gekregen waarin stond hoe de blokken in elkaar zaten. Die blokken heb ik verdeel in een plaatje, tekst en titel. Deze heb ik in de custom fields gezet.



Hierna hebben Jantine, Cyd, Seb en ik een tweede kick-off gehouden. We hebben de opzet van de website af en Jantine, Cyd en ik hebben nieuwe tickets gemaakt. Tijdens het maken van deze stand-up kwamen er veel tickets op mijn naam. In het begin maakte ik me een beetje zorgen want ik dacht dat dit te veel was voor mij.

1

Mijn eerste ticket was om de header te maken. Het begin verliep niet helemaal soepel. Ik deed er minstens twee dagen over om een simpele header te maken. Ik denk dat dit komt door het feit dat ik in de code erg moest zoeken waar alle content stond en omdat deze header ook op mobile versie beschikbaar moet zijn. Uiteindelijk moest ik alles van de oude website verwijderen en opnieuw beginnen.

2

Toen mijn eerste ticket eindelijk klaar was ben ik verder gegaan met de tweede ticket. Dit was het maken van de vacatures. Gelukkig bestond een groot gedeelte van deze blokken al. Het wijzigen van de SCSS vond ik nog wel erg moeilijk aangezien alles werd overschreven, en als je iets wijzigt aan de ene classnaam dan werd het andere element ook aangepast.

Tijdens het maken van deze vacatures moest ik veel classes toevoegen en daarna apart wijzigen in de SCSS. Ik heb nu wel het gevoel dat ik al deze codes beter begrijp aangezien ik dus vooral veel heb gewerkt met de containers en elementen een classnaam en stijl te geven.

3

Hierna ben ik verder gegaan met de events. Ik heb de slider gemaakt met een plugin. Het maken van deze events verliep goed. Ook hier was er al een groot gedeelte gemaakt. Ik hoefde alleen de SCSS wat aan te passen. Dit verliep soepel. Alleen van dit blok een slider maken vond ik erg moeilijk. Matise gebruikt in dit gevoel Slick slider. Dit was een nieuwe plugin. Zelf begreep ik deze plugin niet omdat deze anders werkte dan de swiper plugin die ik gewend ben. De plugin werkte niet in de data en ook niet met Vue dus moest je javascript gebruiken.

Gelukkig heeft Cyd me met dit onderdeel geholpen. Ook moest deze slider pas zichtbaar zijn op de mobiele versie. Hiervoor gebruik je Unslick. De dots aanpassen van deze slider was ook moeilijk. Als je deze dots wilde aanpassen moest je ze een classnaam geven in de javascript. Ik vind dat deze plugin complexer in elkaar zit dan swiper doordat je moet werken in javascript en alles één voor één op true of false functie moet zetten wil je dat dit werkt.



Ik had 20 april een klantgesprek. Wegens de drukke paas dagen was ik dit klant gesprek vergeten. Ik kwam er pas achter onderweg naar werk. Toen ik bij Matise aankwam was het klant gesprek ook al afgelopen. Ik vond dit erg naar omdat ik wel erg uitkeek naar dit klant gesprek en ik wist dat ik hiervan veel zo leren. Gelukkig was er iedere dinsdag een klant gesprek met de opdrachtgever en was ik er de afspraak erna wel gewoon. Jantine zei die dag ervoren dat ik de website mocht presenteren, ik heb tijdens de presentatie kort verteld over de dingen die ik gemaakt heb. De klant was onder de indruk en blij met het resultaat. Na dit gesprek konden we door met het tweede gedeelte van de website.

Hierna heb ik de vacature pagina gemaakt. Wat hier alleen miste waren de plaatjes. Die plaatjes worden met PHP data ingeladen. Als je deze plaatjes in je website wilt hebben moet je debug gebruiken. Cyd heeft uitgelegd hoe debug werkt. Debug is best handig want je kan zien welke PHP data er in je website staat binnen één oog opslag. Het is net een array van data.

Voor het klant gesprek heeft Merijn het design nog gecontroleerd hij vond een paar dingen die ik en Cyd hebben verbeterd.

Eén van die dingen was dat de icons niet op de juist plek stonden. De icons werden ingeladen met PHP data. Wat wel een uitdaging was is om de icons een juiste stijl de geven. Gelukkig heeft Cyd al eens voorgedaan hoe debug werkte. Want bij dit onderdeel kwam het ook van toepassen aangezien ik de classnaam nodig had van deze icons.





Het midden deel twee:

In dit gedeelte ben ik aan de slag gegaan met de content pagina. Dit gedeelte verliep vrijwel soepel. De oorzaak hiervan is denk ik omdat ik nu een beetje weet hoe de code in elkaar zit.

Ik begon met mijn eerste ticket. Dat was om een nieuw content blok aan te maken. Ik heb ook geleerd hoe deze in de php code moest zetten. Er is namelijk een mapje themessrc hiervoor beschikbaar. Hierin staat een mapje partials met daarin nog een mapje genaamd templates. Hierin staan alle blokken dus, de blokken die de klant ook kan gebruiken met custom fields. Ik heb zelf een simpel-content blok aangemaakt.

Ik heb via WordPress custom fields dan een YouTube link erin gezet. Deze heb ik met de SCSS aangepast.

De derde ticket was om block08 aan te passen. Ook dit gedeelte verliep soepel, alleen moest hiervoor een slider gemaakt worden. Hiervoor heb ik ook Slick slider toegepast. Aan de andere kant vond ik slick slider dus erg moeilijk maar al met al was het ook wel weer goed om hier nog eens aan te werken zodat ik het eventueel later beter begrijp. In het begin lukte het echt niet om deze slider te maken ook omdat ik hier gewoon te weinig van begreep. Ik heb precies hetzelfde gedaan als de slider op de homepagina en dan met een andere classnaam maar dit hielp niet.

Ik denk dat ik wel een hele dag bezig was geweest met deze slider omdat het gewoon echt niet lukte. Uiteindelijk heeft Cyd me geholpen. Ik heb dezelfde eigenschappen als de eerste slider gebruikt alleen heeft deze een andere classnaam. Ook heb ik deze dots een andere naam gegeven. Hierna ben ik verder gegaan met de searchbar en de zoekresultaten. De SCSS hiervan aanpassen lukte wel.

Daarnaast moest er nog een pagination komen. Deze heb ik gemaakt de PHP code van WordPress zelf.

'mid_size' => 2,
'prev_text' => __( 'Back', 'textdomain' ),
'next_text' => __( 'Onward', 'textdomain' ), ) ); ?>

Wat wel een uitdaging was is om de preview en next knoppen te vervangen met je eigen stijl. Cyd heeft dit voorgedaan en verteld dat de previous en next een aparte classnaam heeft en dat je deze gewoon zelf kan aanpassen en dat dit ook automatisch wordt overschreven. Ik heb deze previous en next gewijzigd met een classnaam en een before.

Hierna ben ik verder gegaan met de maps. De klant wilde namelijk een pop-up window hebben wanneer er geklikt wordt op één van de roze points. Ik heb de stijl hiervan aangepast. Cyd heeft weer een linkje gevonden waarin een documentatie stond over google maps pop-ups. Ik heb eerst een div aangemaakt met de classnaam pop-up daarna heb ik deze link samengevoegd met de code van de maps. Daarna heeft Seb dit omgezet in PHP code om dit ook te koppelen met de data.

We hadden 20 mei de laatste stand-up gehouden waarin we het lijstje van Jantine en Stephan opnieuw afgingen. In dit lijstje hebben we namelijk duidelijk verteld wat we aan de klant zouden opleveren. Cyd vond nog een paar dingen die niet helemaal in orde waren en die we wel hadden beloofd. De website is 29 mei klaar om getest te worden.
